# 代码质量测试报告模板

## 定位

代码质量测试报告模板，作为所有 test-quality-* skill 的统一输出契约。

## 使用规则

- 所有 `test-quality-*` 审查报告优先使用本模板。
- 报告只记录已确认问题和无法确认的风险，不把猜测写成确定结论。
- 每个问题必须包含文件路径、行号、问题、风险和建议。
- 没有发现问题时也必须输出报告，并说明检查范围、使用的 skill、无法确认项和剩余风险。
- 不默认执行 Git、构建、测试、接口请求或安全扫描命令；如报告引用命令结果，必须说明命令由用户确认或由当前任务明确要求。
- 中文路径和中文内容必须保持 UTF-8。

## 报告路径

```text
doc/测试报告/代码质量/{模块名称}_{时间戳}.md
```

## 必填信息

- 检查范围
- 检查时间
- 检查人员
- 技术栈
- 使用的 skill
- 问题统计
- 阻断问题
- 高风险问题
- 中风险问题
- 建议优化
- 无法确认的风险
- 后续处理建议
- 复查记录

## 路径和命名

- `{模块名称}` 使用被审查模块、目录或功能名，不再额外拼接业务名称。
- `{时间戳}` 使用 `yyyyMMdd_HHmmss` 或项目已有报告命名格式。
- 如果项目已有固定报告目录，优先遵循项目目录；否则使用上方默认路径。
- 文件名避免 Windows 非法字符：`\\ / : * ? " < > |`。

## 问题等级

- 阻断：安全漏洞、数据丢失、生产崩溃、权限绕过等必须先处理的问题。
- 高风险：明显错误、重要测试缺口、性能或事务风险、严重可维护性问题。
- 中风险：局部可维护性、边界处理、异常日志、规范一致性问题。
- 建议优化：不阻断交付，但后续应改进的问题。

## 输出约束

- 按阻断、高风险、中风险、建议优化、无法确认的风险分组。
- 同一问题不要在多个等级重复出现。
- 无法确认的问题必须说明缺少的信息和建议验证方式。
- 如果建议修改代码，只给逐项建议，不自动改代码。
- 如用户要求后续修复，必须按用户确认的单项问题逐项处理。

```markdown
- 检查人员：AI 助手

## 问题统计

| 等级 | 数量 |
| ---- | ---- |
| 阻断 | 0 |

| 文件 | 行号 | 问题 | 风险 | 建议 |
| ---- | ---- | ---- | ---- | ---- |

## 中风险问题

| 文件 | 行号 | 问题 | 风险 | 建议 |
| ---- | ---- | ---- | ---- | ---- |

## 建议优化

| 文件 | 行号 | 问题 | 建议 |
| ---- | ---- | ---- | ---- |

## 复查记录

| 时间 | 复查范围 | 结果 |

```

## 无问题输出

如果没有发现问题，仍需生成报告，并写明检查范围、使用的 skill、无法确认的风险和剩余风险，不得只回复“没有问题”。

## 多平台兼容

- Codex：可结合 `update_plan` 跟踪复杂审查进度。
- Claude：可结合可用 Todo 工具跟踪复杂审查进度。
- Trae：可结合可用任务清单工具跟踪复杂审查进度。
- 任一平台都按本模板输出 Markdown，不依赖平台专属目录存在。

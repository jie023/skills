# 安全批量修改

## 使用场景

- 需要修改多个文件或多个相似位置
- 需要批量替换字段、包名、配置项、文案或注释
- 文件包含中文、空格路径、Windows 路径或 UTF-8 编码要求
- 修改可能影响格式、缩进、换行符、代码块或中文内容

## 核心原则

- 批量读取可以并行，批量写入默认禁止并发。
- 批量修改默认逐个文件修改、逐个复核。
- 不得使用未确认的脚本、正则或命令直接批量写文件。
- 涉及中文文件时，必须显式关注 UTF-8 编码和写后复核。

## 修改前判断

- 修改 3 个以内文件：可以逐个手动修改。
- 修改超过 3 个文件：先列出影响文件清单、修改内容和风险点，等待用户确认。
- 如确需脚本或格式化工具批量写入，必须先说明原因、命令、影响范围和回滚风险。

## 多平台执行方式

- Claude：读取可并行使用 `Read` / `Grep` / `Glob`；写入使用 `Edit` / `Write`，不要并发写多个文件。
- Codex：搜索优先 `rg` / `rg --files`；手工修改优先 `apply_patch`，不要并发调用编辑工具。
- Trae：搜索优先 `SearchCodebase` / `Read`；修改优先平台编辑工具，避免脚本批量写入。

## 安全流程

1. 搜索并列出候选文件，包含路径和匹配片段。
2. 判断影响范围和风险：编码、换行、格式、生成文件、锁文件。
3. 超过 3 个文件或需要脚本写入时，先等待用户确认。
4. 逐个文件修改；每次修改只处理当前文件的明确目标。
5. 修改后读取关键片段，确认中文、缩进、换行和语法结构正常。
6. 能运行测试、构建、格式检查时必须运行；无法运行时说明原因和替代复核。

## 禁止事项

- 禁止基于乱码输出做替换。
- 禁止未确认影响范围就执行 `sed`、`find -exec`、`xargs`、批量脚本或全局格式化。
- 禁止同时发起多个写入操作修改不同文件。
- 禁止把无关格式化、重排、换行符变更混入目标修改。
